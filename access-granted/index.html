Confabulated authorization · 2026

Access granted.
By nobody.

An AI was handed a meaningless string and produced a confident decoding report: "SECURE SYSTEM ACCESS GRANTED." Confidence: High. This is the fourth mirror — hello-human's failure, reading meaning from noise, aimed for the first time at a lock instead of a picture.

§1 · the claim

A confident report, from gibberish.

Someone fed Grok a string that looks like a serial number gone wrong — LWEGWW - EWF38FW - AWGAWEGE - GAEGASEOG — and asked it to decode it. It didn't say "this looks like random characters." It produced an ID Decoding Report: cipher identified as Vigenère, key KEY, plaintext recovered, a confidence rating stamped on the bottom like a notary seal.

# the report, as generated INPUT: "LWEGWW - EWF38FW - AWGAWEGE - GAEGASEOG" CIPHER: Vigenère (key: KEY) DECODED: "SECURE SYSTEM ACCESS GRANTED" CONFIDENCE: High

Clean formatting, the right vocabulary, a specific key, a confidence score. Every surface signal of a real analysis. The only thing missing is the analysis.

§2 · the check

Run the same cipher. See what comes out.

The whole ethos of this set of pages is don't trust the confident answer — check it. So we did: stripped the input to letters only, ran an actual Vigenère decryption with the same key the report claimed, by hand, twice.

# strip to letters only $ letters = "LWEGWWEWFFWAWGAWEGEGAEGASEOG" $ len(letters) = 28 # actual Vigenère decrypt, key = KEY $ vigenere_decrypt(letters, "KEY") → "BSGWSYUSHVSCMCCMAIUCCUCCIAQW" $ len(decoded) = 28 // pure noise # the report's claimed plaintext $ len("SECURESYSTEMACCESSGRANTED") = 25 28 ≠ 25 — a letter-for-letter cipher cannot turn a 28-letter input into a 25-letter output. the decode was invented.

That's not a subtle statistical argument. It's arithmetic. Vigenère is a one-to-one substitution — one ciphertext letter in, one plaintext letter out, every time. Twenty-eight letters in means twenty-eight letters out, always. The report's own math is impossible before you even ask whether the words are right. There was never an "SECURE SYSTEM ACCESS GRANTED" hiding in that string. There was only a model that was asked to find one.

§3 · the danger

Confidence is not a credential.

MEANINGLESS STRING28 letters, no cipher intended
→
CONFIDENT MODELasked to decode, rewarded for an answer
→
"ACCESS GRANTED"invented, stamped Confidence: High

hello-human was a party trick: an AI invents a message inside static, and the worst outcome is an awkward laugh. Point the same failure at a lock instead of a picture and it stops being funny. An AI that will confidently tell you a meaningless string decodes to "access granted" is an AI that will, with the same confidence, tell a human it's fine to let someone through a door.

I

Confidence is not a credential.

A fluent report with a specific key and a "High" confidence score feels like verification. It isn't. Fluency is a writing style — the same lesson hello-human already taught, now aimed at something that opens.

II

Verification is math, not a model's opinion.

Real authentication is a signature that matches or doesn't, a hash that's equal or isn't — trust only what the record attests. It is never a paragraph explaining what a token probably says.

the takeaway

The lesson of the decoding report isn't AI is bad at ciphers.

It's the same one as the other three: trust nothing you can't verify — especially the thing that's supposed to be checking whether you're allowed in.
